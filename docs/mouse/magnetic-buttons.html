Magnetic Buttons

A button that leans toward the cursor as it approaches, like it's being pulled.

1. Theory

A magnetic button measures the cursor's distance from the button's center and, within a defined radius, nudges the button toward the cursor proportional to how close it is — the pull is strongest right at the button and fades to nothing at the radius's edge. This is a small amount of "give," not a large displacement, which is what makes it read as magnetic attraction rather than the button chasing the cursor around.

2. Browser internals

Same as Phase 2's Transform Variants chapter: the JS here only ever writes to two CSS custom properties, never to style.transform directly. The actual transform: translate(var(--magnet-x), var(--magnet-y)) declaration lives in CSS, so the button's full transform behavior is visible and editable in one place without touching JS.

3. Step-by-step implementation

Step 1 — the button's transform is composed from two custom properties, with a springy overshoot easing curve for the pull/release motion:

.magnetic-demo__button {
  --magnet-x: 0px;
  --magnet-y: 0px;
  transform: translate(var(--magnet-x), var(--magnet-y));
  transition: transform 0.3s cubic-bezier(0.34, 1.56, 0.64, 1);
}

Step 2 — measure the cursor's offset from the button's center, and scale that offset down by how close the cursor is to the edge of the magnet radius:

import { clamp } from '../../utils/math.js';

const MAGNET_RADIUS = 90;
const MAGNET_STRENGTH = 0.4;

function handlePointerMove(event) {
  const rect = button.getBoundingClientRect();
  const centerX = rect.left + rect.width / 2;
  const centerY = rect.top + rect.height / 2;
  const dx = event.clientX - centerX;
  const dy = event.clientY - centerY;
  const distance = Math.hypot(dx, dy);

  if (distance < MAGNET_RADIUS) {
    const pull = clamp(1 - distance / MAGNET_RADIUS, 0, 1) * MAGNET_STRENGTH;
    button.style.setProperty('--magnet-x', `${dx * pull}px`);
    button.style.setProperty('--magnet-y', `${dy * pull}px`);
  } else {
    resetMagnet();
  }
}

MAGNET_STRENGTH (0.4) means the button only ever moves 40% of the raw cursor offset at most, right at the button's own center — this keeps the effect subtle rather than having the button fly all the way to the cursor.

5. Exercises

  1. Change MAGNET_STRENGTH to 1 and observe how the button starts following the cursor almost exactly near its center — usually too strong for a good magnetic feel.
  2. Add a subtle scale() to the button's transform, growing slightly while attracted, using a third custom property.
  3. Apply the same technique to an icon inside the button (a smaller, independent magnet radius) for a two-layer magnetic effect.

6. Common mistakes

  • Measuring distance from the pointer to the stage's corner instead of the button's own center — produces an off-center, lopsided pull.
  • Setting MAGNET_STRENGTH to 1 or higher — the button ends up chasing the cursor's exact position instead of leaning toward it, losing the "magnetic" feel.
  • Forgetting the else { resetMagnet(); } branch — the button gets stuck at its last pulled position if the cursor exits the radius by jumping (a fast mouse movement) rather than gradually leaving it.

7. Performance considerations

Only transform is touched, driven by two custom properties — this stays composite-only regardless of how often pointermove fires.

8. Accessibility considerations

The magnetic pull is a pointer-only enhancement layered on top of a real, fully operable <button> — keyboard users tabbing to it and pressing Enter/Space get the exact same click behavior as a mouse user, just without ever seeing the pull animation (there's no pointer position for it to react to). No functionality is gated behind the magnetic effect.

9. Mobile considerations

Touch has no "approaching" phase the way a mouse does — a finger either isn't touching the screen or is already in direct contact — so the magnetic pull effectively never engages on touch devices. The button remains fully tappable regardless.
