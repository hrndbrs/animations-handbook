Gradient Text & Underline Animation

Two decorative text effects, both pure CSS — no JavaScript in this chapter at all.

1. Theory

background-clip: text clips an element's background (which can be a gradient) to the shape of its text glyphs, instead of its box. Combined with color: transparent (so the actual text color doesn't paint over the clipped background), this produces gradient-colored text using only CSS — the same background-position-driven movement technique from Phase 2's Shadows & Gradients chapter, just clipped to text shape instead of a rectangular box.

The animated underline uses a similar background trick: instead of text-decoration, a background-image gradient (here just a solid color) sized to 0% 2px and positioned at the bottom acts as the underline; transitioning background-size to 100% 2px on hover/focus grows it left-to-right.

2. Browser internals

background-clip: text still requires the -webkit- prefix in Safari despite being otherwise standard; both prefixed and unprefixed versions are included here. Since both demos in this chapter only ever animate background-position/background-size (not layout-affecting properties), they stay cheap regardless of text length.

3. Step-by-step implementation

Step 1 — the gradient text: a background gradient wider than the element, clipped to the text, animated via background-position:

.gradient-text-demo__heading {
  background: linear-gradient(90deg, var(--color-accent), var(--color-accent-2), var(--color-accent));
  background-size: 200% auto;
  background-clip: text;
  -webkit-background-clip: text;
  color: transparent;
  -webkit-text-fill-color: transparent;
  animation: gradient-text-shift 3s linear infinite;
}

@keyframes gradient-text-shift {
  from { background-position: 0% 50%; }
  to { background-position: 200% 50%; }
}

Step 2 — the animated underline: a bottom-anchored background gradient, sized to zero width at rest, transitioned to full width on hover/focus:

.underline-demo__link {
  background-image: linear-gradient(var(--color-accent), var(--color-accent));
  background-size: 0% 2px;
  background-repeat: no-repeat;
  background-position: left bottom;
  transition: background-size 0.3s ease;
}

.underline-demo__link:hover,
.underline-demo__link:focus-visible {
  background-size: 100% 2px;
}

Using a background-image underline instead of text-decoration-line: underline is what makes it animatable via background-size — the native text-decoration underline has no equivalent "grow from one side" transition.

5. Exercises

  1. Change the gradient text's animation-timing-function from linear to ease-in-out and compare the perceived motion.
  2. Change the underline's background-position to right bottom and observe it growing from the opposite side.
  3. Add a third color stop to the gradient text's linear-gradient() and observe how it changes the color transition as it scrolls.

6. Common mistakes

  • Forgetting color: transparent alongside background-clip: text — without it, the solid text color paints on top of the clipped gradient and hides it entirely.
  • Forgetting the -webkit- prefix for background-clip: text — Safari still requires it.
  • Using width instead of background-size to animate the underline's growth — animating width on a pseudo-element/span forces layout on every frame, where background-size stays compositor-friendlier.

7. Performance considerations

Both effects animate only background properties, not layout- affecting ones, so they remain cheap regardless of how much text is involved.

8. Accessibility considerations

Neither demo splits or hides any text — color: transparent only affects paint, not the accessibility tree, so screen readers read the gradient heading normally. The underline link has a visible :focus-visible state matching its hover state, so keyboard users get the same affordance mouse users do. Reduced motion is handled by the sitewide rule in shared/doc-page.css, which stops the gradient text's infinite loop the same way it stops Phase 2's Shadows & Gradients chapter's gradient bar — no local override needed here either.

9. Mobile considerations

The gradient heading needs no interaction at all. The underline link's hover state won't trigger on touch devices without a mouse, but its :focus-visible state still applies when reached via keyboard navigation, and the link remains fully functional via tap regardless of whether the visual underline animation plays.
